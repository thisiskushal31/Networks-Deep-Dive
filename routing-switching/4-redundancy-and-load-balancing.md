## HSRP

**HSRP (Hot Standby Router Protocol)** is a Cisco proprietary **first-hop redundancy protocol**. Two or more routers in the same subnet act as one **virtual router**: they share one **virtual IP** and one **virtual MAC** that hosts use as their **default gateway**. Only one router is **active** and forwards traffic; the other(s) are **standby**. If the active fails, a standby becomes active so the default gateway stays available.

**Terms (from source):**

- **Virtual IP** — An IP from the local subnet configured as the default gateway for all hosts.
- **Virtual MAC** — Generated by HSRP. Format: 0000.0c07.ac**XX** (Cisco 24 bits + HSRP ID 07.ac + group number in hex, e.g. 0a for group 10). Example: 0000.0c07.ac0a.
- **Priority** — Default 100. Higher priority wins to become active. When the original active comes back, you can set the current active’s priority lower so the original takes over again.
- **Preempt** — Standby can take over when the active fails; with preempt, when the former active recovers it can take back the active role (if its priority is higher).
- **Hello messages** — Exchanged every **3 seconds** between active and standby.
- **Hold-down timer** — Default about **10 seconds** (3× hello). If hellos stop, the standby waits this long before taking over.

**Working (from source):**

1. Routers in the same **group ID** form one HSRP group.
2. One is elected **active**, the rest **standby**. All use the same virtual IP and MAC.
3. Hosts use the **virtual IP** as default gateway; the **active** router forwards their traffic.
4. If the active fails, hellos stop; after the **hold-down** time the **standby** becomes active (preempt).
5. When the original active returns, you can lower the current active’s priority so the original becomes active again.

**Versions:**

- **Version 1:** Multicast 224.0.0.2, UDP 1985, group 0–255.
- **Version 2:** Multicast 224.0.0.102, UDP 1985, group 0–4095.

**Object tracking:** You can tie HSRP priority to an **object** (e.g. an interface or route). If the tracked object goes down, the router’s priority is reduced so another router in the group becomes active. This gives **failure-based failover** (e.g. uplink down → give up active role).

**Visual flow:**

```text
  Hosts (default gw = 10.1.1.100)
        |
        |  Virtual IP 10.1.1.100, Virtual MAC 0000.0c07.ac01
        v
  +----------+     +----------+
  | R1       |     | R2       |
  | Active   |     | Standby  |
  | 10.1.1.1 |     | 10.1.1.2 |
  +----------+     +----------+
        |
  If R1 fails → R2 becomes Active after hold-down; traffic continues via 10.1.1.100.
```

## GLBP

**GLBP (Gateway Load Balancing Protocol)** is a Cisco proprietary **first-hop redundancy** protocol that adds **load balancing** across multiple routers. Like HSRP, one **virtual IP** is used as the default gateway, but GLBP uses **multiple virtual MACs** (one per router in the group). The **AVG** assigns which MAC each host gets (e.g. round-robin), so different hosts use different physical routers and traffic is spread.

**Terms (from source):**

- **Virtual IP** — One IP from the local subnet, used as default gateway by all hosts.
- **AVG (Active Virtual Gateway)** — One router in the group that answers **ARP** for the virtual IP and assigns **virtual MAC** addresses to the **AVFs**. Elected by priority (or IP if tie). Sends hellos to 224.0.0.102 every 3 seconds.
- **AVF (Active Virtual Forwarder)** — Each router in the group that forwards traffic using the virtual MAC the AVG gave it. If the AVG fails, an AVF with next-highest priority becomes AVG.
- **Object tracking** — GLBP can use a **weight** that is reduced when a tracked interface (or object) goes down, so that router is less preferred for forwarding.
- **Preempt** — When the AVG fails, an AVF can become the new AVG; when the old AVG comes back, it can take over again if priority is higher.

**Load-balancing algorithms (from source):**

1. **Round-robin** — AVG hands out virtual MACs in turn (AVF1, AVF2, …) so traffic is spread.
2. **Weighted** — Traffic share is proportional to configured weight.
3. **Host-dependent** — Same host always gets the same virtual MAC (same AVF).

**Redundancy:**

- **AVG:** If the AVG fails, the AVF with highest priority becomes AVG; it can send **gratuitous ARP** to update switches and hosts.
- **AVF:** If one AVF fails, another in the group can take over its virtual MAC (forwarder redundancy). Up to **4** routers in a GLBP group.

**Visual flow:**

```text
  Host A (gw 10.1.1.100) → ARP → AVG replies with MAC-A → traffic to R1 (AVF1)
  Host B (gw 10.1.1.100) → ARP → AVG replies with MAC-B → traffic to R2 (AVF2)
  ...
  Single virtual IP, multiple virtual MACs → load shared across R1 and R2.
```

## Load balancing and redundancy in L3 design

**First-hop redundancy** means the default gateway (first hop for off-subnet traffic) is **highly available**:

- **HSRP / VRRP** — One virtual IP and one active router; standby takes over on failure. No load balancing of default-gateway traffic (all hosts use the same active router).
- **GLBP** — One virtual IP but multiple virtual MACs; AVG distributes which router each host uses, so you get **redundancy and load sharing** with a single gateway address.

**In L3 design:**

1. Use **HSRP** (or VRRP) when you only need **failover** and a single active gateway is fine.
2. Use **GLBP** when you also want **load balancing** of default-gateway traffic across two or more routers.
3. Use **object tracking** so the active router gives up its role if its uplink (or another tracked object) fails, instead of only when the router itself dies.
